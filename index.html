Key Features
🏢
AWS Organizations Native
SCPs deployed and versioned as code across every OU and account
🔒
Preventive Guardrails
Block non-compliant actions before they happen with mandatory SCPs
🔍
Detective Controls
Detect policy violations in real-time via Config Rules and EventBridge
💰
Cost Guardrails
Enforce budget alerts and resource tagging policies at the account level
🚀
Account Vending
Automate new account creation with baseline security, logging, and IAM via CDK
📊
Compliance Dashboard
Real-time compliance score per account, OU, and framework across the org
Who Is It For?
Cloud Platform Teams
Dev teams create resources without tags or security controls
SCPs and Config Rules enforce tagging and security standards before resources are created
Enterprise Architects
New AWS accounts take weeks to provision securely
Account vending machine delivers fully configured, compliant accounts in under 10 minutes
FinOps Teams
Surprise cloud bills from untagged or unbudgeted resources
Mandatory tagging + account-level budgets with automated alerting catch overspend instantly
Quick Start
$ git clone https://github.com/charlesbulabula/multi-account-governance
$ cd multi-account-governance
$ npm install -g aws-cdk && pip install -r requirements.txt
$ cdk bootstrap aws://MANAGEMENT_ACCOUNT_ID/us-east-1
$ cdk deploy GovernanceStack
